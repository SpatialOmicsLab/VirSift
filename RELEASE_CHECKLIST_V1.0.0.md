# VirSift v1.0.0 Release Checklist

Use this checklist before publishing the first formal VirSift release.

## 1. Repository Preparation

- [ ] `README.md` is at the repository root and renders correctly on GitHub.
- [ ] `LICENSE` contains the MIT License and the correct copyright holder.
- [ ] `CITATION.cff` validates and contains the final version metadata.
- [ ] `CODE_OF_CONDUCT.md` is present.
- [ ] `CONTRIBUTING.md` is present and links to the Code of Conduct.
- [ ] Architecture and logical-workflow diagrams render correctly.
- [ ] Screenshot links open the full-resolution images.
- [ ] The Molecular Timeline screenshot is readable at full resolution.
- [ ] Python, Streamlit, license, language, version, and DOI-placeholder badges render.
- [ ] Repository description, website, topics, and social preview are configured.
- [ ] Repository website points to `https://virsift.streamlit.app/`.
- [ ] No placeholder such as `<YOUR-ACCOUNT>` or `DOI pending` remains unintentionally.

## 2. Naming, Scope, and Claims

- [ ] The public product name is consistent: `VirSift` or `Vir-Seq-Sift`.
- [ ] Repository, app interface, screenshots, diagrams, citation, and release title use the agreed name.
- [ ] The scientific scope is accurate: influenza-focused or broader viral FASTA support.
- [ ] Claims about supported header formats, languages, charts, and sampling strategies match the released code.
- [ ] Local-versus-hosted processing wording is accurate.
- [ ] Charts are described as sequence-dataset summaries, not epidemiological case counts.

## 3. Codebase

- [ ] `app.py` starts without errors on Python 3.11.
- [ ] Every page under `pages/` loads correctly.
- [ ] No hardcoded local paths remain.
- [ ] No credentials, API keys, private URLs, or local environment files are committed.
- [ ] No leftover debug prints or temporary test controls remain.
- [ ] The original pre-filter dataset remains immutable after activation.
- [ ] Translation fallback works.
- [ ] All six released catalogues contain the same 817 keys.
- [ ] Translation keys load from all released language files:
  - [ ] Arabic
  - [ ] English
  - [ ] Spanish
  - [ ] French
  - [ ] Russian
  - [ ] Chinese
- [ ] `requirements.txt` installs successfully in a clean virtual environment.

## 5. Test Data and Compliance

- [ ] Every file in `cases/` has documented provenance.
- [ ] Every public test file is synthetic or independently redistributable.
- [ ] No restricted GISAID-derived record is included without express authorization.
- [ ] NCBI/GenBank accession and provenance information are retained.
- [ ] README disclaimers render correctly.
- [ ] App footer disclaimer is implemented.
- [ ] Hosted deployments do not invite upload of restricted or sensitive data without appropriate safeguards.

## 6. Automated Checks

- [ ] `black --check .` passes.
- [ ] GitHub Actions lint workflow passes.
- [ ] Automated tests pass.
- [ ] No workflow uses `pytest || echo "No tests yet"` or another command that hides failures.
- [ ] Manual test results are recorded for areas without automated coverage.

## 7. Packaging and Versioning

- [ ] Version number is `1.0.0` in the application and README.
- [ ] Final release date is added to `CITATION.cff`.
- [ ] `CHANGELOG.md` or release notes summarize the first public release.
- [ ] Git tag is created as `v1.0.0`.
- [ ] GitHub release title is `VirSift v1.0.0`.
- [ ] Source archives generated by GitHub are reviewed.
- [ ] Optional supplementary files are attached to the release.

## 8. Zenodo DOI

- [ ] Zenodo GitHub integration is enabled for the repository.
- [ ] The GitHub `v1.0.0` release is published.
- [ ] Zenodo creates the version DOI.
- [ ] The DOI is added to the README badge and citation text.
- [ ] The DOI is added to `CITATION.cff`.
- [ ] The Zenodo metadata and author order are verified.

## 9. Paper and Documentation Preparation

- [ ] Paper figures are exported at publication-appropriate resolution.
- [ ] Figure legends state that sequence counts are not case counts.
- [ ] The correct statement from `PAPER_DATA_AVAILABILITY.md` is selected.
- [ ] GISAID acknowledgment and author-attribution requirements are satisfied.
- [ ] NCBI/GenBank accessions are listed where applicable.
- [ ] Software version and DOI are stated in the manuscript.
- [ ] The repository is public at the time required by the journal or archive.
